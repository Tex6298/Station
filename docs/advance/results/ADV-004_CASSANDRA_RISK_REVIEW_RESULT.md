## Risk Ordering

1. Demo Caveat And Freshness Checklist
   - Can become shadow PR319 hosted-freshness work, a deploy verdict, or a
     stale demo-proof claim if it goes beyond a no-probe template.
2. Anonymous Chat Risk Checklist
   - Can imply anonymous access is under active consideration or approved, and
     can pull rate limits, identity, abuse handling, transcripts, reports, and
     provider cost into an unpromoted lane.
3. Partner And Commercial Readiness Questions
   - Can turn into commercial positioning before entitlement, support, billing,
     pricing, and pilot ownership are actually decided.
4. External Pilot Decision Form
   - Can read like an invitation plan if it names audiences, dates, or success
     claims before MIMIR/Marty names a real pilot.
5. Config And Request Inventory For Future Pilots
   - Safe only while categories-only. It becomes harmful if it asks for values,
     account identifiers, deploy access, or live operational evidence.
6. Public Persona Answer-Quality Fixture Template
   - Useful later, but can accidentally create a shadow model-quality bar or
     tempt prompt/completion capture if repeated-answer evidence is not already
     safely bounded.
7. Public Launch Claim Checklist
   - Safest because it reduces overclaim risk without needing live systems,
     but it must not become launch approval, copy editing, or public
     positioning.
